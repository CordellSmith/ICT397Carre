Apply post-processing to an already-imported scene.

This is strictly equivalent to calling \hyperlink{cimport_8h_37baf75d55599334097f7337ce8f25c5}{aiImportFile()}/\hyperlink{cimport_8h_60c8f08f9daa728b5e2d20623e81cd13}{aiImportFileEx} with the same flags. However, you can use this separate function to inspect the imported scene first to fine-tune your post-processing setup. \begin{Desc}
\item[Parameters:]
\begin{description}
\item[{\em pScene}]Scene to work on. \item[{\em pFlags}]Provide a bitwise combination of the \hyperlink{postprocess_8h_64795260b95f5a4b3f3dc1be4f52e410}{aiPostProcessSteps} flags. \end{description}
\end{Desc}
\begin{Desc}
\item[Returns:]A pointer to the post-processed data. Post processing is done in-place, meaning this is still the same \hyperlink{structai_scene}{aiScene} which you passed for pScene. However, \_\-if\_\- post-processing failed, the scene could now be NULL. That's quite a rare case, post processing steps are not really designed to 'fail'. To be exact, the \hyperlink{postprocess_8h_64795260b95f5a4b3f3dc1be4f52e410e420ce22fbbac9d0fd21fd92f2b630fa}{aiProcess\_\-ValidateDataStructure} flag is currently the only post processing step which can actually cause the scene to be reset to NULL. \end{Desc}
